# Отчет по практике: Интеграция программных модулей информационной системы с использованием API

**Наименование работы:** Интеграция программных модулей информационной системы с использованием API

**Цель задания:** Получение практического опыта разработки и интеграции программных модулей с использованием программного интерфейса приложений (API).

## 1. Архитектура и реализация системы

Разработанная информационная система состоит из двух ключевых модулей, взаимодействующих по принципу "клиент-сервер" через REST API.

### 1.1. Серверный модуль (API-провайдер)
*   **Технология:** Python, фреймворк **FastAPI**.
*   **Функционал:** Предоставление программного интерфейса для получения, передачи и обработки данных (в данном случае — управление списком задач/элементов).
*   **Обеспечение корректности:** Использование Pydantic для строгой валидации входящих и исходящих данных.

### 1.2. Клиентский модуль (API-потребитель)
*   **Технология:** Python, библиотека **CustomTkinter**.
*   **Функционал:** Графический интерфейс пользователя (GUI) для отправки запросов к API, отображения полученных данных и обработки ответов.
*   **Интеграция:** Взаимодействие с API осуществляется через библиотеку `requests`.

## 2. Описание API

Разработанный API является **RESTful** и использует формат обмена данными **JSON**.

| Метод | Endpoint | Описание | Функциональное требование |
| :---: | :--- | :--- | :--- |
| **GET** | `/items` | Получение полного списка элементов. | Получение данных |
| **GET** | `/items/{item_id}` | Получение информации о конкретном элементе по его ID. | Получение данных |
| **POST** | `/items` | Создание нового элемента в системе. | Передача данных |
| **DELETE** | `/items/{item_id}` | Удаление элемента по его ID. | Обработка запросов |

## 3. Описание структуры данных

Для обеспечения логической целостности и корректности форматов данных используется модель `Item`, определенная с помощью Pydantic.

**Модель данных `Item` (JSON-формат):**

```json
{
  "id": 1,
  "title": "Наименование задачи",
  "description": "Подробное описание задачи",
  "status": "active"
}
```

| Поле | Тип данных | Описание |
| :--- | :--- | :--- |
| `id` | `integer` | Уникальный идентификатор элемента (генерируется сервером). |
| `title` | `string` | Краткое название элемента. |
| `description` | `string` | Детальное описание. |
| `status` | `string` | Текущий статус элемента (например, "active", "completed"). |

## 4. Примеры запросов и ответов

### 4.1. Получение данных (GET /items)

**Запрос:**
```http
GET /items HTTP/1.1
Host: 127.0.0.1:8000
```

**Ответ (HTTP 200 OK):**
```json
[
  {
    "id": 1,
    "title": "Пример задачи 1",
    "description": "Описание первой задачи",
    "status": "active"
  },
  {
    "id": 2,
    "title": "Пример задачи 2",
    "description": "Описание второй задачи",
    "status": "completed"
  }
]
```

### 4.2. Передача данных (POST /items)

**Запрос:**
```http
POST /items HTTP/1.1
Host: 127.0.0.1:8000
Content-Type: application/json

{
  "title": "Новая задача",
  "description": "Необходимо интегрировать модуль"
}
```

**Ответ (HTTP 200 OK):**
```json
{
  "id": 3,
  "title": "Новая задача",
  "description": "Необходимо интегрировать модуль",
  "status": "active"
}
```

### 4.3. Обработка ошибочных запросов (GET /items/999)

**Запрос:**
```http
GET /items/999 HTTP/1.1
Host: 127.0.0.1:8000
```

**Ответ (HTTP 404 Not Found):**
```json
{
  "detail": "Item not found"
}
```

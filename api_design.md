# Проектирование информационной системы

## Архитектура системы
Система состоит из двух основных модулей:
1.  **Серверный модуль (Server)**:
    *   Реализован на Python с использованием фреймворка **FastAPI**.
    *   Предоставляет REST API для управления данными (в данном примере — каталог товаров/задач).
    *   Обрабатывает GET, POST запросы.
    *   Обеспечивает валидацию данных и обработку ошибок.
2.  **Клиентский модуль (Client)**:
    *   Реализован на Python с использованием библиотеки **CustomTkinter**.
    *   Взаимодействует с сервером через HTTP-запросы (библиотека `requests`).
    *   Визуализирует полученные данные и предоставляет формы для отправки новых данных.

## Структура API (Endpoints)
*   `GET /items` — Получение списка всех объектов.
*   `GET /items/{item_id}` — Получение информации о конкретном объекте.
*   `POST /items` — Добавление нового объекта.
*   `DELETE /items/{item_id}` — Удаление объекта.

## Формат данных (JSON)
Пример объекта:
```json
{
  "id": 1,
  "title": "Наименование",
  "description": "Описание",
  "status": "active"
}
```
